Linux 以數字順序批次改檔名並前綴補 0

在用數字順序排檔名時，前綴補 0 可以讓檔案排序更整齊方便，也比較好處理，下面範例是之前我有用過的方法，分別使用了 rename.ul 與 Shell Script

使用 rename.ul

這是 rename.ul man page 裡的 example，也算是常用到的情況

將 A1、A2… ~ A100 改為 A001、A002… ~ A100

rename.ul A A0 A?

rename.ul A A0 A??

rename.ul 的用法為：rename.ul 原字串 替換字串 要換的檔案

在第一個指令中，先把 A 後面接一個字元的檔案（A?，如 A1 ~ A9） 的 A 改成 A0，因此 A1 ~ A9 就會變成 A01 ~ A09

接著第二個指令再把 A 後面接兩個字元的檔案（A??，A01 ~ A99）也多個 0，就變成 A001 ~ A099

使用 Shell Script

一樣是將 A1 ~ A100 改成 A001 ~ A100，可以用一行 shell script 來處理

ls -v | cat -n | while read line file; do mv "$file" A$(printf "%03d" $line); done

這個方法主要是用了 cat -n 印出的行號來做迴圈處理，建議先一步一步的測試會比較理解這個方法，最後用 echo 來測試檔案輸出的結果，確定後再正式使用

  • ls -v：使用正常的數字大小排序
  • cat -n：將前面排序好的列表加上行號
  • while read line file：以迴圈分別將行號與檔名存到變數 line 與 file
  • printf “03d% $line”：檔名如果不到3個字元，前面就用 0 來補齊，如果是 1 ~ 100 最多就是三個字，因此用 “03d%"，如果是 1 ~ 1000 那就是用 “04d%” 了

這邊要注意的是如果 ls -v 得到的順序不是我們想要的，那就要用 sort 依自己的需求再做一次排序，後面再接上 cat -n

本來還想寫用 seq 指令先做出數列的 list，再讀取 list 作為檔名來更名的方法，不過沒有上面用 script 一行搞定來得簡潔，所以類似的方法就之後再提了。

另外我時常會有要將音樂檔重命名的需求，不過目前都用 mp3tag 來做 leading zero 的步驟，使用 shell 的方法暫時還不知道（雖然一張專輯的曲目也沒有多到一定得用批次處理）
